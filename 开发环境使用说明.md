# AI智能网关开发环境使用说明

## 完全容器化开发（代码挂载模式）

### 特点
- ✅ 代码直接挂载到容器中
- ✅ 无需每次构建镜像
- ✅ 支持热重载
- ✅ 环境一致性
- ✅ 快速启动

### 快速开始

#### 1. 启动开发环境
```bash
# 方式1：快速启动
start_dev_fast.bat

# 方式2：使用管理工具
dev_manage.bat
```

#### 2. 访问地址
- **前端**: http://localhost:3000
- **后端API**: http://localhost:8001
- **数据库**: localhost:3307
- **Redis**: localhost:6379

### 开发流程

#### 前端开发
1. 修改 `frontend/src/` 下的代码
2. 浏览器自动刷新（热重载）
3. 无需重启容器

#### 后端开发
1. 修改 `config-center/` 下的代码
2. 后端自动重启（热重载）
3. 无需重启容器

### 常用命令

#### 查看日志
```bash
# 前端日志
docker-compose -f docker-compose.dev.yml logs -f frontend

# 后端日志
docker-compose -f docker-compose.dev.yml logs -f config-center

# 所有日志
docker-compose -f docker-compose.dev.yml logs -f
```

#### 进入容器
```bash
# 前端容器
docker exec -it ai-gateway-frontend-dev sh

# 后端容器
docker exec -it ai-gateway-config-center-dev bash

# 数据库容器
docker exec -it ai-gateway-mysql-dev bash
```

#### 安装依赖
```bash
# 前端依赖
docker exec -it ai-gateway-frontend-dev npm install

# 后端依赖
docker exec -it ai-gateway-config-center-dev pip install -r requirements.txt
```

#### 重启服务
```bash
# 重启前端
docker-compose -f docker-compose.dev.yml restart frontend

# 重启后端
docker-compose -f docker-compose.dev.yml restart config-center

# 重启所有服务
docker-compose -f docker-compose.dev.yml restart
```

### 开发技巧

#### 1. 代码修改
- 直接修改本地代码文件
- 容器会自动检测变化并重载
- 无需重新构建镜像

#### 2. 依赖管理
- 前端：修改 `package.json` 后运行 `npm install`
- 后端：修改 `requirements.txt` 后运行 `pip install -r requirements.txt`

#### 3. 调试
- 使用浏览器开发者工具调试前端
- 查看容器日志调试后端
- 进入容器进行深度调试

#### 4. 数据库操作
- 使用MySQL客户端连接 localhost:3307
- 用户名：ai_gateway，密码：ai_gateway_pass
- 数据库：ai_gateway_config

### 故障排除

#### 1. 端口冲突
- 检查端口是否被占用
- 修改 `docker-compose.dev.yml` 中的端口映射

#### 2. 依赖问题
- 重新安装依赖
- 清理容器缓存

#### 3. 热重载不工作
- 检查文件权限
- 重启容器

#### 4. 数据库连接问题
- 检查数据库容器状态
- 查看数据库日志

### 环境清理

#### 完全清理
```bash
# 停止并删除所有容器和卷
docker-compose -f docker-compose.dev.yml down -v

# 清理Docker系统
docker system prune -f
```

#### 部分清理
```bash
# 只停止容器
docker-compose -f docker-compose.dev.yml down

# 重启特定服务
docker-compose -f docker-compose.dev.yml restart [service-name]
```

### 注意事项

1. **代码挂载**：确保代码文件权限正确
2. **依赖安装**：首次启动会自动安装依赖
3. **端口占用**：确保端口3000、8001、3307、6379未被占用
4. **文件同步**：Windows下文件同步可能有延迟
5. **资源使用**：容器会占用一定系统资源

### 性能优化

1. **使用SSD**：提高文件同步速度
2. **增加内存**：提高容器性能
3. **关闭杀毒软件**：减少文件监控影响
4. **使用WSL2**：在Windows下获得更好性能
